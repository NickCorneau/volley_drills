# PR Title

`feat(ui): consolidate primitive layer into ui/ + patterns/ split`

# PR Body

## Summary

Eight inline UI patterns scattered across `app/src/screens/` and `app/src/components/` are now centralised behind shared primitives. Two structural smells the audit surfaced are fixed alongside (`ActionOverlay`'s string-typed focus contract; surface tokens co-located with `Card`). A custom ESLint rule fails at edit time when the consolidated patterns get hand-rolled inline again.

- **Origin:** [`docs/brainstorms/2026-05-04-ui-primitive-consolidation-requirements.md`](docs/brainstorms/2026-05-04-ui-primitive-consolidation-requirements.md)
- **Plan:** [`docs/plans/2026-05-04-008-feat-ui-primitive-consolidation-plan.md`](docs/plans/2026-05-04-008-feat-ui-primitive-consolidation-plan.md)
- **Net delta:** +3754 / -924 across 56 files
- **In-flight coordination:** absorbs and supersedes the in-flight `2026-05-04-007-feat-app-wide-choice-ui-consistency-plan.md`

## What changed

### Folder split

- `components/ui/` — atomic primitives (Button, Card, ChoiceRow, ChoiceSection, ChoiceSubsection, Disclosure, Expander, Callout, NumberCell, ToggleChip, ScreenShell, ActionOverlay, StatusMessage, surfaces).
- `components/patterns/` — Volleycraft-shaped composites (ScreenHeader, RunFlowHeader, ConfirmModal, JustFinishedPill, BackButton).
- `components/ui/index.ts` re-exports the moved patterns for back-compat so no call-site import paths break.

### New primitives

| Primitive | Replaces |
|---|---|
| `ChoiceRow` | 8+ inline `<div role="radiogroup">` wrappers across SetupScreen, SafetyCheckScreen, TuneTodayScreen, RpeSelector, IncompleteReasonChips, PerDrillCapture |
| `ScreenHeader` | Hardcoded `BackButton + h1 + w-12 spacer` trio in SetupScreen, SafetyCheckScreen, SettingsScreen, TuneTodayScreen |
| `RunFlowHeader` | 3-cell grid eyebrow header in RunScreen, TransitionScreen, DrillCheckScreen (40-line "why grid not flex" comment now lives once) |
| `JustFinishedPill` | Verbatim copy-pasted warm-panel + success-circle pill in DrillCheckScreen + TransitionScreen |
| `Disclosure` | Two duplicated "Add X (optional)" reveal buttons in PerDrillCapture |
| `Expander` | SafetyCheckScreen heat tips chevron expander |
| `ConfirmModal` | Title + safe-primary + destructive-secondary modal shape in SoftBlockModal, SkipReviewModal, RunScreen end-session sheet |
| `Callout` | Inline tone-driven panels in SafetyCheckScreen heat warning + tips, PainOverrideCard, SettingsScreen export-success, StatusMessage error body |
| `NumberCell` | Parallel implementations in `PassMetricInput.NumberCell` + `PerDrillCapture.StreakInput` |

### Structural moves

- **`surfaces.ts` extraction (S1)** — `FOCAL_SURFACE_CLASS` + `ELEVATED_PANEL_SURFACE` move out of `Card.tsx` (where they were a leaky abstraction — non-Card surfaces had to import from a Card module). `Card.tsx` re-exports for back-compat.
- **`ActionOverlay` typed-focus seam (S2)** — replaces the `data-action-overlay-initial-focus="true"` string-attribute contract with a typed `initialFocusRef?: RefObject` prop. Migrated all 5 callers. Eliminates a TypeScript-uncheckable bug class (typo'd attribute = silent first-focusable fallback).
- **Folder split (S3)** — `components/patterns/` houses Volleycraft-shaped composites; `components/ui/` keeps atomic primitives.

### Drift guardrail

Custom local ESLint rule `volleycraft/no-inline-primitive-drift` (in `app/eslint-rules/`) fails at edit time on:

1. `role="radiogroup"` outside `ChoiceRow.tsx` → use `<ChoiceRow>`.
2. `data-action-overlay-initial-focus` anywhere → removed in U2; pass `initialFocusRef` instead.

`RuleTester` suite locks both checks. `.cursor/rules/component-patterns.mdc` updated with the new primitive names + Drift Guardrail section.
